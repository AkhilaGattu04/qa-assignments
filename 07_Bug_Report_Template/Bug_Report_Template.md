## Severity Level Guide

| Severity | Definition | Examples |
|----------|-----------|---------|
| **Critical** | Complete system failure, data loss, security breach, incorrect payment sent | Application crashes on invoice submit; vendor bank account exposed; duplicate payment processed |
| **High** | Core functionality broken with no workaround | Cannot log in; cannot submit invoice; AP team cannot approve/reject |
| **Medium** | Feature works incorrectly but a workaround exists | Rejection email sent without the rejection reason; report shows wrong totals |
| **Low** | Minor issues not affecting core functionality | Misaligned UI element; incorrect error message wording; tooltip missing |
| **Cosmetic** | Purely visual with no functional impact | Font size inconsistency; border color mismatch |
